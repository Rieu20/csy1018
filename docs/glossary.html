HTML GLOSSARY

HTML stands for Hyper Text Markup Language, is the standard language used to create all webpages. It describes the content of a webpage semantically

This is the basic tag that defines an html document. The rest of the html file goes in the html tag

Attributes

Attributes contain extra information about the element which you don't want to appear in the actual content

Article

Article tag specifies self independent, self contained content.

Article element should make sense on itsown and should be possible to distribute it independtly from the rest of the site.

Basic Formatting

Text can be formatted to be bold, italic, and underlined using simple formatting tags

Body

The body tag defines the document body. This element contains all the contents of an HTML document such as text, hyperlinks, images, tables, list. It comes after the head tag within the overall tag

Class

HTML classes can have one or more clases seperated by spaces. Elements can be syled using CSS by selecting them with their classes

Children

An element that is an instant descendant of another element or nested within another element is called a child. These becomes useful in CSS child sectors

Comments

HTML comments are used in the code to explain part of the markup.They are similar to comments in other language. Comments are not displayed by the brower but they can help document your HTML source code

Directory

The is a computer file folder oraganised as a hierarchy.The root directory is called "htdocs" that includes all other directories, these are conventions in programming

Div

This is a block level container or division of the web page for content with no semantic meaning. The div element is very often used together with CSS, to layout a web page.By default, browsers always place a line break before and after the element. However, this can be changed with CSS.

DOCTYPE

The DOCTYPE declaration must be the very initial thing in any HTML document, before the html tag. It specifies the rules for the mark-up language, so that the browsers render the content correctly.

Tip: Always add the declaration to your HTML documents, so that the browser knows what type of document to expect.

Flex Box

The Flexible Box Layout Module (Flexbox) is an API that provides tools to make web content responsive. Flexbox provides an efficient way to lay out, align, and distribute space among items in a container, even when their size is unknown and/or dynamic. The API allows the rapid creation of complex, flexible layouts, and features that have historically proved difficult with CSS.

Footer

A footer defines a footer for a document or section .

Head

Tag that surrounds important content that is invisible to the user, but is important to the browser. Elements within this tag contain metadata about the page and links to stylesheets, scripts, etc.

This is the basic tag that defines an html document. The rest of the html file goes in the html tag

HTML Responsive Web Design

Responsive Web Design is about using HTML and CSS to resize,hide, shrink, enlarge, or move the content to make it look good on all devices (desktops,laptops and phones)

HTML Validator

An HTML Validator is a quality assurance programm used to check Hypertext Markup Language (HTML) markup elemnts for syntax errors

Headings

Heading elements allow you to use six levels of document headings, ranging from largest to smallest, breaking up the document into logical sections. Example h1 defines the most important heading, h6 defines the least important heading

Hyperlinks

These links take the user into another webpage when click on. Most common attribute used with links is a href which tells the browser where the link goes

Horizontal Rules

This tag creates a black line one pixel thick that run all the way across its container.The hr tag defines a thematic break in an HTML page (e.g. a shift of topic). This element is also used to separate content (or define a change) in an HTML page. This can be styled differently in CSS

Href

A link that tells the browser where to go using an href attribute, which stores a URL.

ID

HTML element have an id attribute to identify it. Id elements should always be unique to that single element, and each element should never have more than one id.

Images

The img tag embeds an image into your HTML. Always found with the 'src' attribute, which tells the browser where to find the image. Images can improve the appearance and design of the web page. The tag is self-closing..

Line Breaks

This line is used in a block of text to force a line break. It is normally used to for things such as which are single paragraphs but where formatting or editing is necessary such as poems or addresses.

To seperate paragraphs, distinct each paragraph into seperate element.

Links

Link elements are used to join your document to a related resource (very different from hyperlinks, which take you to another webpage when you click on them). Links appear only in the head section of a document so they do not change the content, but only the presentation. Links are most normally used to connect to a stylesheet, script, favicon, or substitute format of the page such as an RSS feed or PDF

List

A list is a sequence of paragraphs, each of which may be proceeded by a special mark or sequence number. HTML supports two kinds of lists namely ordered lists and unordered lists. Within lists each individual list item has its own tag

Unordered lists are just lists whose items are denoted with bullet points.

Ordered lists' items are denoted with numbers.

Paragraphs

This is one of the most common tags in HTML, it denotes a paragraph of text. It often has other elements nested inside of it

Properties

Properties are defined within selectors

Semantic Formatting

These tags are like formatting tags which have fallen out of favour. The difference is that the tags have semantic value (meaning). It is used for something that you wish to emphasize on and strong is used considered important. With both elements, you can take the level of emphasis with nesting. The more times that you nest the element within itself, the higher the scale of the text it contains.

Syntax

These are the sets of rules that defines the combinations of symbols that are correctly structured document or fragment in the language

Tags and Elements

Tags are basic labels that define and separate parts of your markup into elements. They are comprised of a keyword surrounded by angle brackets. Content goes between two tags and the closing one is prefixed with a slash (Note: there are some self-closing HTML tags, like image tags). Tags also have attributes, which are attributes

Title

This tag tells the browser what to display as the page title at the top and tells search engines what the title of your site is. It goes inside head tags. Try and make your page titles descriptive, but not overly verbose.

Table

An element for displaying information in rows and columns. Supports headers and footers for labelling columns. Divides information into rows (denotated by the tag ) which contain cells (denoted by the td tag )

Padding

The Padding is the spacing between the content and the border (edge of the element. The value can be adjusted with CSS to move the border closer to or farther from the content.

Selectors
